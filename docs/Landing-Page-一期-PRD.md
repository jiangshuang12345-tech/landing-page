## 2. 整体业务流程

从广告点击到 ROI 闭环的完整用户与数据流转路径：

```text
广告点击 (带渠道码) 
  ↓
Landing Page (网页)
  ↓
注册 / 登录 (手机号 / 第三方)
  ↓
选择套餐 (按国家定价)
  ↓
应用优惠码 (选填)
  ↓
选择支付方式并完成支付
  ↓
引导下载 App / App 内 VIP 自动生效
```

## 3. 详细交互需求（逐屏走查）

### 3.1 注册 / 登录页 (Sign up)
> **视觉更新重点**：增加品牌信息：13年品牌信任背书，地球与全球连线设计，突出No.1 AI英语App定位。

- **品牌与营销文案**（需多语言）：
  - 核心Slogan：“The No.1 AI English App for Kids — 13 Years of Trust, Smart AI Tutors”
  - 副标题：“Making English learning part of your child's daily route”
- **注册方式 1（首位突出）：手机号**
  - 支持 4 国手机号（沙特/印尼/马来/韩国），国家码下拉可选。
  - 验证码逻辑与 App 内一致：点击「Get code」发送并倒计时 60s，倒计时中退出返回后继续；结束后变「Resend」。
  - 验证码满 4 位后 [Log in] 按钮由置灰变亮起可点击；提交正确后登录成功。
- **注册方式 2：第三方一键登录**
  - google / apple / facebook / kakao (韩国特有) 一排小图标，一键拉起第三方登录。
- **其他细节**：
  - 隐私协议 / 用户条款：4 国多语言，均可跳转链接，放置于登录按钮正下方。
  - 默认姓名与 App 内一致（后端预置 100 个姓名随机给出）。

**异常状态**：
- `手机号格式错误`: Please input the valid phone number
- `网络异常`: Network error. Please try again
- `请求过于频繁 (5 分钟最多 5 次)`: Too many requests. Please try again later
- `验证码错误`: Incorrect verification code
- `验证码过期 (有效期 5 分钟)`: Verification code expired. Please request a new one

### 3.2 支付 / 套餐页 (Dino Pro)
> 按国家展示套餐、价格、币种、折扣，默认高亮 Best Value。

- 文案全部需多语言（Dino Pro 除外）：权益、套餐+价格、政策链接、支付说明「支付后 App 内立即生效，可随时取消」。
- 页面地址需一直携带 `国家` + `渠道码` 参数。
- 点击返回可回到注册页，允许修改手机号或重新拉起第三方授权。
- 所有套餐均为一次性购买；按国家展示不同套餐 / 价格 / 币种 / 折扣标签。
- 默认选中并高亮带「Best Value」标签的套餐。
- 点击「Continue」跳转优惠码页面。

### 3.3 优惠码页 (Apply coupon)
> 一码多人兑换，限时有效，入库记录原价与折后价。

- **无优惠码**：仅展示套餐名（如 1 year）+ 价格，例如总计 $229.99。
- **有优惠码**：展示原价 229.99 + 直减 40（高亮）+ 总计 $189.99。
- **入库要求**：订单数据需同时记录 原价 与 折扣后价格。
- **交互逻辑**：
  - 输入前 Apply 按钮置灰；输入后亮起可点击校验。
  - 校验成功：按钮变绿 + 勾选 + 显示优惠文案（例如 You saved $40.00）。
  - 点击返回上一页支持修改优惠码信息。

**异常状态**：
- `网络异常`: Network error. Please try again
- `优惠码错误`: Incorrect promotion code
- `优惠码过期`: Promotion code expired

### 3.4 选择支付方式 (Payment)
> 按国家展示支持的支付渠道，点击 Pay now 拉起第三方支付。

- **支持渠道**：
  - 韩国：NaverPay、TossPay、SamsungPay、PayCo、Kakaopay、银行卡
  - 印尼 / 马来 / 沙特：银行支付
- 点击「Pay now」拉起对应的第三方支付 SDK 或收银台。
- 返回按钮可回上一页重新输入优惠码。
- 除支付方式名称外，其它文案均需多语言。

**异常状态**：
- `支付成功`: 跳转支付成功页面
- `支付失败`: 支付失败，请重试（留在当前页）
- `取消支付`: 支付已取消，请重试（留在当前页）

### 3.5 支付成功页 (Payment Complete)
> 展示已购套餐与有效期，引导下载 App 用同账号登录学习。

- 无论用户此前是否在 App 内购买过，在 Web 端支付成功都直接展示本成功页。
- 页面展示：恭喜文案、套餐名+价格、VIP 有效期、核心权益列表。
- 引导去 App 用同一账号登录学习；提供客服邮箱 `support@dinoenglish.ai`；提示 VIP 立即生效。
- **「Start Learning」按钮动作**：
  - 若已下载 App：拉起并打开 App
  - 若未下载 App：跳转 Google Play 或 iOS App Store
- **VIP 权益发放在后端的逻辑**：
  - 同账号立即发放；如果用户在 App 端用不同账号登录则不发放，需联系客服人工处理。
  - 有效期叠加规则：非会员 = 套餐到期时间；已经是会员 = 在当前 Google / App Store 订阅的有效期基础上往后顺延叠加。

### 3.6 返回主页 / 复访 (Home)
> 根据用户当前的登录与购买状态，决定复访落地页时的重定向落点。

- **登录失败**：返回初始注册页。
- **登录成功但未购买 / 购买失败**：第二次进入重新展示注册/登录（保持一致）。
- **登录 + 购买成功后（首页及后续复访）**：直接展示支付成功页信息，方便用户随时查询已购套餐的状态。

## 4. 埋点 & 数据架构

**设计原则**：
GA4 负责 Web + App 行为漏斗；AppsFlyer 负责安装归因与渠道追踪；UserID/UID 在 Web 登录后与 App 打通；AppsFlyer 推荐使用 OneLink + Smart Script 做 Web-to-App 传递。

**漏斗节点**：
1. `广告点击`：URL 参数携带 `ch = 7 位渠道码`。
2. `Landing Page`：触发 GA4 `page_view`。
3. `校验渠道码`：服务端提取 Session Country + Channel。
4. `登录 / 注册`：Phone OTP / OAuth 成功后触发 GA4 `login` 并带上 `channel_code`。
5. `支付环节`：成功触发 GA4 `purchase` + `channel_code`；失败则回退。
6. `AppsFlyer OneLink`：Web ↔ App UID 绑定，用户安装 App 激活后，AF 确认安装归因。
7. `自动 VIP`：Firebase / GA4 触发 `first_open` / `subscribe` / `purchase`。
8. `CRM 入库`：记录 `channel_code` / `country` / `UID`，状态更新为 Paid → Activated。
9. `ROI 闭环`：最终归因数据回传至对应的广告平台。

## 5. 渠道码总表

**生成规则**：
格式为固定的 7 位随机字符串，字符集采用大小写字母 + 数字，并去除易混淆字符（0, O, o, 1, l, I, i）。
运营/CMS 预先为「国家 × 广告渠道」生成唯一码存入映射表；广告链接仅带 `?channel=xxxxxxx`，落地页反查获得所属国家与渠道。

目前 4 国 × (6 广告渠道 + 1 默认) = 共 28 个预置渠道码，详情请查看在线交互式 PRD 或配置后台字典。

## 7. 退款机制与流程

退款规则通过各国的政策链接展示，不在支付页面直接呈现。对外统一客服口径：审核通过后 14 天内原路退回到账。

**计算公式：**
1. **购买后 ≤ 2 天**：可退 = 实付金额（例：第1天申请，退100%）
2. **购买后 > 2 天**：可退 = 实付 × (剩余天数 / 总天数)
   *(例：月包实付 99.90，第 10 天申请退款，计算 99.90 × 20/30 = 66.60)*
3. **使用 Coupon 时**：以最终实付金额为基数（券后价参与计算，优惠券不予退回）
4. **SKU 总天数定义**：周包 = 7，月包 = 30，年包 = 365。剩余天数 = 总天数 − 已购天数（下限为 0）。

**审批处理流：**
1. 用户联系客服申请退款。
2. 客服核实订单，计算可退金额，人工发起退款邮件/工单。
3. 直系领导审批确认。
4. 技术团队介入取消对应 UID 的 VIP 权益，将 CRM 状态从 `Activated` 变更为 `Revoked`。
5. 财务确认后执行退费，调用对应支付网关的 Refund API。
6. 用户收到退款（14天内）。
